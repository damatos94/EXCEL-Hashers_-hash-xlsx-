Что это такое

Полная реализация алгоритма хэширования MD5 внутри Excel-файла — без VBA, без макросов, без плагинов. Только стандартные функции INT, MOD, MID, CODE. Введи слово — хэш пересчитывается мгновенно.

Структура файла

MD5
Главная страница. Вводишь слово в жёлтую ячейку — внизу появляется хэш. Всё остальное пересчитывается само.

Как работает MD5
Пошаговое объяснение алгоритма: паддинг, 4 раунда, функции F/G/H/I, константы K, финализация.
🔤
ASCII-кодирование
Каждый символ слова → ASCII-код → отдельные биты б7..б0. Видно как текст превращается в числа.
📦
Константы MD5
Таблица всех 64 констант K[i] (floor(2³²·|sin(i+1)|)) и 64 значений сдвига r[i] по раундам.
⚙️
Вычисление
Сердце файла: 64 строки компрессии. Каждая строка — один шаг MD5 с промежуточными значениями BA1/BA2/BA3 → ROL → new_B.

04

Алгоритм MD5 кратко

MD5 берёт произвольное сообщение и выдаёт 128-битный «отпечаток». Даже одна изменённая буква полностью меняет хэш.

Раунд 1 · F
F(B,C,D) = AND(B,C) + AND(¬B, D)
B выбирает C или D
Раунд 2 · G
G(B,C,D) = AND(B,D) + AND(C, ¬D)
вариант выбора
Раунд 3 · H
H(B,C,D) = XOR(XOR(B,C), D)
побитовая чётность
Раунд 4 · I
I(B,C,D) = XOR(C, OR(B, ¬D))
нелинейное смешивание
КАЖДЫЙ ШАГ
new_B = ADD32( B, ROL32( ADD32(A, F/G/H/I, M[g], K[i]), r[i] ) )
где M[g] — 32-битное слово сообщения, K[i] — синусоидальная константа, r[i] — величина сдвига
05

Как биты считаются без BITOR/BITAND

В Excel нет нативного 32-битного целочисленного типа. Все побитовые операции реализованы через чистую арифметику:

AND32(a, b)
разбиваем на 4 байта × 8 бит через INT/MOD, суммируем совпадающие биты
OR32(a, b)
= a + b − AND32(a, b)
XOR32(a, b)
= a + b − 2 × AND32(a, b)
NOT32(a)
= 4294967295 − a
ROL32(x, n)
= MOD( MOD(x, 2^(32−n)) × 2^n + INT(x / 2^(32−n)), 2³² )
ADD32(a, b)
= MOD(a + b, 4294967296)

07

Ограничения

⚠️
Максимум 7 символов — из-за структуры MD5-блока (512 бит = 64 байта). При более длинном слове паддинг потребует нескольких блоков, что умножит число формул. Файл поддерживает один 512-битный блок — достаточно для учебных целей.

Только ASCII — кириллица и Unicode дают многобайтовые символы, которые не совпадают с CODE().

Excel 2013+ — нужны функции INT, MOD, MID, CODE в их современных формах. LibreOffice Calc не поддерживается в полной мере.

Не для production — скорость пересчёта ~1–3 сек на слабых машинах из-за 947 формул.
