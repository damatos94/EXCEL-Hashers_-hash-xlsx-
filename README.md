# 🔐 MD5 на формулах Excel

> Полная реализация алгоритма MD5 внутри `.xlsx` файла — без VBA, без макросов, только стандартные формулы Excel.

```
"hello"  →  5d41402abc4b2a76b9719d911017c592
"abc"    →  900150983cd24fb0d6963f7d28e17f72
"test"   →  098f6bcd4621d373cade4e832627b4f6
```

---

## Как пользоваться

1. Открыть файл `MD5_Pure_Formulas.xlsx` в **Excel 2013+**
2. Перейти на лист **🔐 MD5**
3. Ввести слово (до 7 символов ASCII) в жёлтую ячейку `B5`
4. Хэш появится в зелёной ячейке ниже — автоматически

## Как устроен алгоритм

MD5 принимает сообщение и выдаёт 128-битный хэш. Даже одна изменённая буква полностью меняет результат.

**Подготовка сообщения:**
1. Каждый символ → ASCII-байт
2. После последнего байта добавляется `0x80`, затем нули до длины ≡ 56 (mod 64)
3. В байты 56–63 записывается исходная длина в битах (little-endian)
4. Итого: ровно 512 бит = 16 слов по 32 бита `M[0]..M[15]`

**Начальное состояние:**
```
A₀ = 0x67452301
B₀ = 0xEFCDAB89
C₀ = 0x98BADCFE
D₀ = 0x10325476
```

**64 операции — формула каждого шага:**
```
new_B = ADD32( B,  ROL32( ADD32(A, F/G/H/I(B,C,D), M[g], K[i]),  r[i] ) )
new_A = D,  new_C = B,  new_D = C
```

**Функции по раундам:**

| Раунд | Операции | Функция | Смысл |
|-------|----------|---------|-------|
| 1 | 1–16   | `F(B,C,D) = AND(B,C) + AND(¬B, D)` | B выбирает C или D |
| 2 | 17–32  | `G(B,C,D) = AND(B,D) + AND(C, ¬D)` | вариант выбора |
| 3 | 33–48  | `H(B,C,D) = XOR(XOR(B,C), D)` | побитовая чётность |
| 4 | 49–64  | `I(B,C,D) = XOR(C, OR(B, ¬D))` | нелинейное смешивание |

**Финализация:** после 64 шагов прибавляем начальные константы (mod 2³²):
```
A_итог = A_64 + A₀  (mod 2³²)
B_итог = B_64 + B₀  (mod 2³²)
...
```
Результат: `A_итог || B_итог || C_итог || D_итог` в little-endian = 32 hex-символа.

## Как побитовые операции реализованы без BITOR/BITAND

В Excel нет 32-битного целочисленного типа. Всё сделано через `INT` и `MOD`:

| Операция | Формула в Excel |
|----------|-----------------|
| `AND32(a, b)` | разбивка на 4 байта × 8 бит через `INT`/`MOD`, суммирование совпадающих битов |
| `OR32(a, b)` | `= a + b − AND32(a, b)` |
| `XOR32(a, b)` | `= a + b − 2 × AND32(a, b)` |
| `NOT32(a)` | `= 4294967295 − a` |
| `ROL32(x, n)` | `= MOD( MOD(x, 2^(32−n)) × 2^n + INT(x / 2^(32−n)),  2³² )` |
| `ADD32(a, b)` | `= MOD(a + b, 4294967296)` |

Функция `AND32` — базовая. Пример для одного байта (8 бит):
```
AND_byte(a, b) = Σ  MOD(INT(a/2^k), 2) × MOD(INT(b/2^k), 2) × 2^k,  k = 0..7
```
Для 32-битного числа это делается по 4 байтам и складывается с весами 1, 256, 65536, 16777216.

## Ограничения

- **Максимум 7 символов** — файл обрабатывает один 512-битный блок. Для более длинных слов нужны дополнительные блоки (и в разы больше формул)
- **Только ASCII** — кириллица и другие многобайтовые символы не поддерживаются
- **Excel 2013+** — нужны современные формы `INT`, `MOD`, `MID`, `CODE`. LibreOffice Calc не поддерживается
- **Не для production** — это учебный материал, скорость пересчёта ~1–3 сек из-за объёма формул
